FlashAttention is an IO-aware exact attention algorithm that dramatically reduces memory usage and accelerates transformer training and inference. Instead of materializing the full NxN attention matrix in slow HBM, FlashAttention tiles computation to keep data in fast SRAM — achieving linear memory complexity and significant throughput improvements.

## What makes FlashAttention fast

Standard attention requires O(N²) memory because it materializes the full attention matrix in HBM. FlashAttention uses tiling to avoid this — kernels load blocks of Q, K, V tiles into SRAM and fuse the softmax+matmul operations in a single pass.

## Get started

<Steps>
  <Step title="Install the package">
    ```bash
    pip install flash-attn --no-build-isolation
    ```
  </Step>
  <Step title="Import and call flash attention">
    ```python
    from flash_attn import flash_attn_func
    import torch

    # q, k, v: (batch, seqlen, nheads, headdim)
    q = torch.randn(2, 1024, 16, 64, dtype=torch.float16, device="cuda")
    k = torch.randn(2, 1024, 16, 64, dtype=torch.float16, device="cuda")
    v = torch.randn(2, 1024, 16, 64, dtype=torch.float16, device="cuda")

    out = flash_attn_func(q, k, v, causal=True)
    # out: (batch, seqlen, nheads, headdim)
    ```
  </Step>
  <Step title="Use in your model">
    Drop FlashAttention into any transformer layer — it is a drop-in replacement for `torch.nn.functional.scaled_dot_product_attention`.
    See the [Guides](/guides/basic-attention) for integration patterns.
  </Step>
</Steps>

## Key capabilities

- **Causal masking** — Auto-regressive language model attention with `causal=True`
- **Sliding window attention** — Local attention with `window_size=(left, right)`
- **GQA / MQA** — Grouped-query and multi-query attention for efficient inference
- **KV cache** — Incremental decoding with `flash_attn_with_kvcache`
- **Variable-length sequences** — Packed batches with `flash_attn_varlen_func`
- **ALiBi** — Attention with linear biases for length extrapolation
- **Softcapping** — Used in Gemma-2 and Grok models
- **torch.compile()** — Compatible with PyTorch 2.4+ compilation
